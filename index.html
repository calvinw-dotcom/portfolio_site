Nommo

Nommo2 is a Django-based web application that provides a Text-to-Speech (TTS) service. Users can input URLs, and the system generates MP3 audio files from the text using backend logic. It features user authentication, a dashboard, and request tracking. Built due to frustration that I could not read news articles in the gym, this webapp was my chance to build on my understanding python and put my, at the time, newly acquired knowledge of Django to use.

This project was also an opportunity to experiment with AI and learn not only its capabilities but, more importantly, its limitations, particularly regarding hardware limitations and biases of the creators. As I chose to ride the deep-seek “wave”, this tool may not accurately represent certain articles, particularly those relating to a certain student-led protest in 1989. Also, as this developed on a laptop with not so spectacular graphics card, the is a bit of a delay in receiving the requested article MP3 file.

Version one used the BeautifulSoup library to scrape the article and return the distilled content. I initially optimised this for BBC news articles, for which I had received permission from the bbc. Once I had this running, I tried using AI to allow the application to scrape text regardless of the HTML structure. However, due to various restrictions in sites’ robots.txt files, this proved difficult to implement. As such, I pivoted to make Nommo2 a news summariser as well as a TTS app

Python, Django, HTML, CSS, JS

During my first few weeks as a tax advisor, I was asked by a Director to calculate the income tax and national insurance contributions of a list of a client’s employees. As this as a time consuming and repetitive task, I saw it as an opportunity to put me new coding skills to use and automate.

Originally, this was a simple script but as I developed my understanding of python and best practices, I went back and split the logic into functions. Looking back, I would have sped up the task by reading directly from the excel file containing the necessary information and writing the PAYE and NIC contributions back to that same file. However, I have chosen not to make any further changes so I can preserve my code to look back on.
